Présentation du projet

Dans le cadre du module Programmation Procédurale, j'ai développé un simulateur de distributeur automatique en langage C. Ce programme simule l'interaction complète entre un client (achat de boissons) et un administrateur (gestion de la machine, des stocks et des recettes).

Rôle

Développeur Backend / Algorithmique

Période

Mai 2026

Outils

Langage C, GCC, VS Code, Git

Objectif

Concevoir un programme simulant un distributeur automatique intelligent permettant :

  • ✔️ Aux clients d’acheter des boissons via une interface simple
  • ✔️ À un administrateur de gérer les ventes et les paramètres via un espace sécurisé

Stack technique & concepts appliqués

🔹 Logique de contrôle : do...while, switch 🔹 Persistance : fichiers (FILE*, fopen, fwrite, fscanf) 🔹 Sécurité : hachage simple / stockage auth.txt 🔹 Algorithmique : rendu de monnaie, synchronisation

Mise en œuvre d'une gestion rigoureuse des types flottants pour les transactions monétaires et d'une architecture modulaire (séparation logique métier / interface / accès aux données).

Fonctionnalités clés

Expérience Client

  • Menu interactif avec choix de boissons
  • Insertion de pièces (0.10€, 0.20€, 0.50€, 1€, 2€) avec validation
  • Calcul du montant restant / rendu de monnaie optimal
  • Incémentation des compteurs et mise à jour du chiffre d'affaires
  • Sauvegarde automatique dans vente.dat

Back-office Administrateur

  • Accès sécurisé par code 999 + authentification (login/mot de passe)
  • Premier lancement : création des identifiants (stockés dans auth.txt)
  • Consultation du chiffre d'affaires global et des statistiques par produit
  • Réinitialisation des données / modification du mot de passe

Fonction de rendu de monnaie (algorithmique gloutonne) :

void rendreMonnaie(float montant) {
    int cents = (int)(montant * 100);
    int pieces[] = {200, 100, 50, 20, 10};
    const char *noms[] = {"2€", "1€", "0.50€", "0.20€", "0.10€"};
    printf("Monnaie rendue :\n");
    for(int i = 0; i < 5; i++) {
        int nb = cents / pieces[i];
        if(nb > 0) {
            printf("%d x %s\n", nb, noms[i]);
            cents %= pieces[i];
        }
    }
}

Bilan personnel & compétences acquises

Défis techniques : La persistance des données a été le challenge majeur : synchroniser les écritures dans vente.dat et auth.txt sans corruption, tout en gérant les accès concurrents (lecture/écriture).

Compétences validées :

  • Manipulation avancée des fichiers en C (flux binaires et textes).
  • Rigueur financière : gestion des flottants pour les euros (conversion en centimes).
  • Conception logicielle modulaire et séparation des responsabilités.

Améliorations futures : Rendre le programme totalement dynamique (ajout/suppression de boissons via fichier de configuration externe) pour éviter la recompilation.
